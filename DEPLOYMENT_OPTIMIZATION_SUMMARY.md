| `monitor.sh`           | 实时监控仪表板       | ✅ 已实现 |
| `benchmark.sh`         | 性能基准测试         | ✅ 已实现 |

#### ✅ 架构设计

```
┌─────────────────┐
│   Nginx (LB)    │
│  Port 80/443    │
└────────┬────────┘
         │
    ┌────┴────┐
    │         │
    ▼         ▼
┌───────┐ ┌───────┐
│ Blue  │ │ Green │
│ 3000  │ │ 3001  │
└───────┘ └───────┘
```

**工作原理**:

1. Blue 活跃，Green 待机
2. 新版本部署到 Green
3. 健康检查验证 Green
4. Nginx 切换流量到 Green
5. Blue 变为备份

**关键特性**:

- ✅ 零停机部署
- ✅ 自动健康检查（30次重试）
- ✅ 自动回滚（失败时）
- ✅ 容器备份保留

## 🛠️ 使用指南

### 自动部署

```bash
# 1. 推送代码
git push origin main

# 2. GitHub Actions 自动执行
# - CI 检查
# - 构建 Docker 镜像
# - 蓝绿部署
# - 健康检查和验证
```

### 手动部署

```bash
# SSH 到服务器
ssh root@7zi.com

# 执行蓝绿部署
cd /root/7zi-frontend
./scripts/deploy/blue-green-deploy.sh auto [image-tag]
```

### 监控部署

```bash
# 实时监控仪表板
./scripts/deploy/monitor.sh

# 部署验证
./scripts/deploy/verify-deploy.sh production

# 性能基准测试
./scripts/deploy/benchmark.sh
```

### 回滚操作

```bash
# 一键回滚
./scripts/deploy/rollback.sh

# 强制回滚（跳过确认）
./scripts/deploy/rollback.sh --force
```
