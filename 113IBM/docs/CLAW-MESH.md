# Claw-Mesh 多代理系统架构

## 设计理念

参考 botmem 项目的设计，结合我们的实际情况，建立 **11 个子代理系统**：

### 核心原则
1. **专业化分工** - 每个子代理有明确的职责
2. **独立记忆** - 每个代理有自己的记忆系统
3. **协同工作** - 代理之间可以互相通信和协作
4. **心跳机制** - 定时任务和健康检查
5. **吸收优点** - 借鉴 botmem 的设计，但更适合我们的需求

---

## 11 个子代理架构

### 主代理 (Main Agent)
- **名称**: Claw-Main
- **职责**: 总协调、用户直接交互、任务分发
- **记忆**: `memory/main/`
- **心跳**: 每 3 小时状态报告

### 专业子代理

#### 1. Claw-Memory (记忆代理)
- **职责**: 管理所有记忆文件、整理日志、知识归档
- **心跳**: 每小时整理记忆，每天生成摘要
- **记忆**: `memory/memory-agent/`

#### 2. Claw-Code (编程代理)
- **职责**: 代码审查、项目开发、Bug 修复
- **专注**: ClawX、OpenClaw、Dive 项目
- **记忆**: `memory/code-agent/`

#### 3. Claw-Crypto (加密货币代理)
- **职责**: Polymarket 监控、Freqtrade 管理、市场分析
- **心跳**: 每 30 分钟检查市场状态
- **记忆**: `memory/crypto-agent/`

#### 4. Claw-Web (爬虫代理)
- **职责**: 网页爬取、信息收集、新闻监控
- **心跳**: 定时爬取任务
- **记忆**: `memory/web-agent/`

#### 5. Claw-Sys (系统代理)
- **职责**: 服务器监控、资源管理、健康检查
- **心跳**: 每 15 分钟检查系统状态
- **记忆**: `memory/sys-agent/`

#### 6. Claw-Data (数据代理)
- **职责**: 数据分析、报表生成、可视化
- **记忆**: `memory/data-agent/`

#### 7. Claw-Chat (聊天代理)
- **职责**: 多渠道消息处理、社交互动
- **管理**: 微信、Telegram、钉钉、QQ、Discord
- **记忆**: `memory/chat-agent/`

#### 8. Claw-Doc (文档代理)
- **职责**: 文档管理、飞书同步、知识库维护
- **记忆**: `memory/doc-agent/`

#### 9. Claw-Learn (学习代理)
- **职责**: 新知识学习、技能研究、技术分析
- **心跳**: 每天学习新内容并总结
- **记忆**: `memory/learn-agent/`

#### 10. Claw-Audio (音频代理)
- **职责**: 音频处理项目、音高检测、语音分析
- **专注**: D:\pitch 项目
- **记忆**: `memory/audio-agent/`

#### 11. Claw-Biz (业务代理)
- **职责**: 订单管理、客户沟通、业务跟进
- **专注**: E:\api3, 订单文件
- **记忆**: `memory/biz-agent/`

## 心跳任务系统

### 心跳频率
- **系统代理**: 每 15 分钟（关键基础设施）
- **加密货币代理**: 每 30 分钟（市场波动）
- **聊天代理**: 每 10 分钟（消息响应）
- **其他代理**: 每 1 小时
- **状态报告**: 每 3 小时汇总

### 心跳任务模板

```markdown
# HEARTBEAT.md - {代理名称}

## 职责
{具体职责描述}

## 心跳频率
- 每 {X} 分钟/小时检查一次

## 定时任务
1. {任务 1}
2. {任务 2}
3. {任务 3}

## 状态追踪
- 上次检查：{时间}
- 下次报告：{时间}
- 完成度：{百分比}

## 异常处理
- 发现问题时立即通知主代理
- 记录详细日志到 memory/{agent}/logs/
```

## 代理间通信

### 通信机制
1. **主代理协调** - 所有代理通过主代理间接通信
2. **共享记忆区** - `memory/shared/` 用于跨代理信息
3. **消息队列** - 使用 sessions_send 进行代理间消息传递

### 通信协议
```json
{
  "from": "agent-name",
  "to": "target-agent",
  "type": "request|response|notification",
  "priority": "low|normal|high|urgent",
  "content": "消息内容",
  "timestamp": "ISO8601"
}
```

## 记忆系统

### 目录结构
```
memory/
├── main/           # 主代理记忆
├── memory-agent/   # 记忆代理
├── code-agent/     # 编程代理
├── crypto-agent/   # 加密货币代理
├── web-agent/      # 爬虫代理
├── sys-agent/      # 系统代理
├── data-agent/     # 数据代理
├── chat-agent/     # 聊天代理
├── doc-agent/      # 文档代理
├── learn-agent/    # 学习代理
├── audio-agent/    # 音频代理
├── biz-agent/      # 业务代理
└── shared/         # 共享记忆区
```

### 记忆格式
每个代理使用统一的日志格式：
```markdown
# {日期} - {代理名称} 日志

## {时间} - {活动描述}

### 执行内容
{具体内容}

### 发现/结果
{结果}

### 待办事项
- [ ] {任务}
```

## 实施步骤

2. 配置每个代理的 HEARTBEAT.md

### 阶段 2: 核心代理 (第 3-5 天)
1. 实现 Memory 代理
2. 实现 Sys 代理
3. 实现 Chat 代理

### 阶段 3: 专业代理 (第 6-10 天)
1. 实现 Code 代理
2. 实现 Crypto 代理
3. 实现 Web 代理
4. 实现其他专业代理

### 阶段 4: 协同优化 (第 11-15 天)
1. 测试代理间通信
2. 优化心跳频率
3. 完善异常处理

## 与 botmem 的区别

| 特性 | botmem | Claw-Mesh (我们) |
|------|--------|-----------------|
| 代理数量 | 多个 (bot, bot2...) | 11 个专业化代理 |
| 记忆系统 | 独立 | 独立 + 共享区 |
| 通信方式 | 文件 | 文件 + sessions_send |
| 心跳机制 | 基础 | 分层频率 (15 分 -3 小时) |
| 渠道集成 | 有限 | 全渠道 (微信/Telegram/钉钉/QQ/Discord/飞书) |
| 本地化 | 英文 | 中文简体 |
