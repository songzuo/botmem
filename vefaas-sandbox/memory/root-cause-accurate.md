- Evomap进程过载 (>20实例)
- SSH连接问题

## 精确根源分析

### 实际观测数据

### 真正的问题

**1. bot (moltbot机器) - 角色混淆**

| 问题 | 说明 |
|------|------|
| 运行moltbot-gateway | 这是交易网关，不是OpenClaw |
| 无OpenClaw | 只有moltbot在运行 |
| picoclaw启动失败 | 模型配置错误 (doubao-seed-code-preview-251028不存在) |

**2. bot5 - SSH不稳定**

| 问题 | 说明 |
|------|------|
| 之前SSH超时 | 云服务商防火墙/网络波动 |
| 现在正常 | 30%磁盘，3个进程 |
| picoclaw已运行 | 端口18795正常 |

### 结论

原分析**不准确**：
- ❌ 磁盘没有87%，只有25-30%
- ❌ 没有Evomap进程过载
- ✅ SSH连接确实不稳定

### 真正需要修复的

1. **bot**: 修复picoclaw配置 (模型错误)
2. **bot/bot5**: SSH连接不稳定 (网络问题，非机器本身)
