## 问题分析

### 根本原因

1. **部署问题**: v1.8.0 未成功部署，服务器仍在运行 1.3.0 版本
2. **进程管理混乱**: 
   - 有一个独立的 `next-server` 进程直接运行在端口 3000
   - PM2 管理的 `picoclaw` 进程因端口冲突无法启动
3. **API 路由问题**: 部分 API 端点不存在或配置错误

### 建议修复步骤

1. **停止独立进程**:
   ```bash
   kill 130476  # 停止独立的 next-server 进程
   ```

2. **部署 v1.8.0**:
   ```bash
   cd /web/7zi
   git pull origin main  # 或正确的分支
   npm install
   npm run build
   pm2 restart picoclaw
   ```

3. **验证端口**:
   ```bash
   lsof -i :3000  # 确保只有一个进程监听
   ```

4. **检查 API 路由**:
   - 确认 `/api/projects`, `/api/tasks`, `/api/agents/status` 是否已实现
   - 检查 Next.js API Routes 配置
