# 自动记忆与任务追踪系统

## 设计目标
1. 任务断点续传 - 知道之前做了什么，是否完成
2. 指令覆盖追踪 - 后面的指令覆盖前面
3. 自动记忆加载 - 会话重启后自动恢复上下文

---

## 实现方案

### 1. 会话启动自动加载 (在 AGENTS.md 中规定)

```
Every Session (必须执行):
1. 读取 SOUL.md
2. 读取 USER.md
3. 读取 memory/2026-MM-DD.md (今天)
4. 读取 memory/2026-MM-DD.md (昨天)
5. 读取 MEMORY.md (长期记忆)
6. 检查 TASKS.md (任务断点)
7. 检查 INSTRUCTIONS.md (最新指令)
```

### 2. 任务断点文件 (TASKS.md)

```markdown
# 任务断点 - 自动生成

## 当前进行中的任务
- [ ] 任务描述 | 优先级 | 开始时间 | 状态

## 最近完成的任务
- [x] 任务描述 | 完成时间

## 待处理任务
- [ ] 任务描述 | 优先级
```

### 3. 指令覆盖记录 (INSTRUCTIONS.md)

```markdown
# 动态指令 - 后面的覆盖前面的

## 最新指令 (2026-03-15 01:50)
### 技能分析
- 2026-03-14: 安装技能 → 2026-03-14: 分析后暂不需要 ✅

### GITHUB_TOKEN
- 2026-03-14: 建议配置 → 2026-03-14: 直接执行完成 ✅
```

### 4. 自动记忆更新 (cron job)

每小时自动执行：
1. 从 sessions.json 提取最近活动
2. 更新 TASKS.md 任务状态
3. 更新 INSTRUCTIONS.md 指令历史

---

## 文件结构

```
memory/
├── TASKS.md          # 任务断点（自动更新）
├── INSTRUCTIONS.md   # 指令覆盖记录（自动更新）
├── 2026-03-15.md    # 每日工作日志
└── ...
```

---

## 使用流程

### 会话启动时
1. 读取 TASKS.md → 知道要继续什么
2. 读取 INSTRUCTIONS.md → 知道最新指令是什么
3. 读取 today.md → 知道今天做了什么

### 任务状态变化时
1. 更新 TASKS.md
2. 追加到 today.md
3. 更新 INSTRUCTIONS.md (如果指令有变化)

### 指令覆盖时
1. 在 INSTRUCTIONS.md 中记录新指令
2. 标注覆盖关系
3. 更新 TASKS.md 任务状态
