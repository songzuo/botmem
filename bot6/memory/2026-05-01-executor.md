- 但 npm 和 pnpm lock 文件同时存在会造成混淆

### 3. 解决 lock 文件冲突 ✅

**已修复**：删除了 `package-lock.json`，只保留 `pnpm-lock.yaml`

```bash
rm -v package-lock.json
```

理由：
- package.json 中有 `pnpm.overrides` 配置，明确指定使用 pnpm
- pnpm-lock.yaml 存在且更小（符合 pnpm 节省空间的特性）
- pnpm 是项目指定的包管理器
