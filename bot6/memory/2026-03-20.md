**发现的问题**:

2. ⚠️ **密钥轮换机制缺失**: 没有 JWT 密钥轮换机制

### 2. 会话处理和安全性

**令牌机制**:
- **访问令牌 (Access Token)**: JWT，有效期 1 小时 (3600秒)
- **刷新令牌 (Refresh Token)**: 数据库存储的随机令牌，有效期 2 小时或 7 天 (取决于 rememberMe 选项)

**Cookie 设置** (`src/lib/api/utils.ts`):
```typescript
// 访问令牌 Cookie
httpOnly: true
secure: process.env.NODE_ENV === 'production'
sameSite: 'lax'
maxAge: 3600  // 1 小时
path: '/'

// 刷新令牌 Cookie
httpOnly: true
secure: process.env.NODE_ENV === 'production'
sameSite: 'lax'
maxAge: 86400 * 7  // 7 天 (rememberMe=true)
或 3600 * 2  // 2 小时 (rememberMe=false)
path: '/'
```

**安全评估**:
- ✅ **httpOnly**: 防止 XSS 攻击窃取令牌
- ✅ **生产环境启用 secure**: 防止中间人攻击
- ⚠️ **SameSite='lax'**: 可能存在 CSRF 风险（考虑改为 'strict'）
- ✅ **双重令牌验证**: JWT + 数据库令牌验证，双重保障
- ✅ **令牌撤销**: 支持单个令牌撤销和全部令牌撤销
- ✅ **密码更改强制重新登录**: 修改密码后撤销所有令牌

**数据库会话表结构**:
```sql
user_tokens (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  token TEXT UNIQUE NOT NULL,        -- JWT 令牌
  refresh_token TEXT UNIQUE NOT NULL, -- 刷新令牌
  expires_at TEXT NOT NULL,
  refresh_expires_at TEXT NOT NULL,
  created_at TEXT NOT NULL,
  last_used_at TEXT,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
)
```

**会话验证流程**:
1. 提取 Authorization header 中的 Bearer token
2. 验证 JWT 签名和过期时间
3. 查询数据库验证令牌存在性和状态
4. 检查用户状态是否为 active
5. 更新 last_used_at 时间戳

**角色定义** (`Role` enum):

**权限系统**:
- **命名规范**: `resource:action` 格式 (如 `user:read`, `task:create`)
- **权限类别**:
  - 用户权限: `user:read/create/update/delete/manage:role`
  - 团队权限: `team:read/create/update/delete/add:member/remove:member/manage`
  - 任务权限: `task:read/create/update/delete/batch/assign`
  - 设置权限: `settings:read/update/manage`
  - 审批权限: `approval:read/create/update/delete/approve/reject/manage`
  - 报表权限: `reports/export/view/manage`
  - 系统权限: `system:read/manage/config`
  - 日志权限: `logs:read/export`
  - AI 智能体权限: `agent:read/create/update/delete/manage/execute`
  - 钱包权限: `wallet:read/manage/transfer`

**多角色支持**:
- ✅ 用户可分配多个角色
- ✅ 权限自动合并（多个角色的权限并集）
- ✅ 支持自定义权限 (`customPermissions`)

**权限检查中间件**:
```typescript
withUserAuth()           // 基础认证
withPermissions(...)     // 需要所有指定权限
withAnyPermission(...)   // 需要任一指定权限
withRole(Role.ADMIN)     // 需要特定角色
withAnyRole(...)         // 需要任一角色
withAdmin()              // 管理员检查
withManagerOrAdmin()     // 经理或管理员
withOptionalAuth()       // 可选认证
```

**权限上下文**:
```typescript

  permissionContext: {
    userId,
    roles,

```

**安全性**:

### 4. 密码存储和安全性

**密码哈希算法** (`src/lib/auth/repository.ts`):
```typescript
crypto.pbkdf2Sync(password, salt, 10000, 64, 'sha512')
```

**密码哈希格式**: `salt:hash`
- Salt: 16 字节随机值 (hex 编码)
- 迭代次数: 10,000 次
- 输出长度: 64 字节
- 算法: SHA-512

**密码强度要求**:
```typescript
- 最少 8 字符
- 至少 1 个大写字母
- 至少 1 个小写字母
- 至少 1 个数字
```

**密码重置功能**:
- ✅ 独立的密码重置令牌表
- ✅ 令牌有效期 1 小时
- ✅ 使用后自动删除
- ✅ 重置后撤销所有活动令牌
- ✅ 不泄露用户是否存在 (安全最佳实践)

**安全评估**:
- ✅ 使用 PBKDF2-SHA512 (安全但较慢)
- ✅ 随机 Salt (每个密码不同)
- ✅ 足够的迭代次数 (10,000)
- ⚠️ **未使用 Argon2 或 bcrypt** (现代更安全的替代方案)
- ✅ 密码强度验证
- ✅ 密码更改强制重新登录
