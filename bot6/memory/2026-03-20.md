## 🔐 Security Fixes

### JWT Secret Hardcoding - FIXED
**Issue**: JWT_SECRET environment variable had hardcoded fallback values, creating a critical security vulnerability.

**Files Fixed**:
- `/root/.openclaw/workspace/7zi-project/src/lib/auth/service.ts`
- `/root/.openclaw/workspace/7zi-project/src/lib/crypto/index.ts`

- `/root/.openclaw/workspace/7zi-project/src/lib/agents/repository-optimized-v2.ts`

**Changes**:
- Removed all hardcoded fallback values (`'default-jwt-secret-key-change-in-production'`, `'default-agent-secret-key'`)
- Updated functions to throw `Error` if `JWT_SECRET` or `AGENT_ENCRYPTION_SECRET` environment variables are not set
- Added JSDoc `@throws` documentation to clarify error conditions
- Production environments now MUST set these environment variables

**Impact**: Prevents use of weak default secrets in production; ensures cryptographic operations fail securely if secrets are missing.

---

### CORS Configuration - FIXED
**Issue**: WebSocket server and some API routes allowed wildcard origins (`*`), enabling cross-site attacks.

**Files Fixed**:
- `/root/.openclaw/workspace/7zi-project/src/app/api/a2a/jsonrpc/route.ts`
- `/root/.openclaw/workspace/7zi-project/src/lib/websocket/server.ts`

**Changes**:
- Changed `Access-Control-Allow-Origin` from `'*'` to strict origin validation
- Now uses `NEXT_PUBLIC_SITE_URL` environment variable (fallback: `'https://7zi.studio'`)
- Updated Socket.IO CORS configuration to use the same strict origin policy
- Added explanatory comments documenting the security requirement

**Impact**: Prevents unauthorized cross-origin requests; only the configured domain can access the API and WebSocket endpoints.

---

### Cookie Security - ENHANCED
**Issue**: Authentication cookies used `SameSite=lax` instead of `SameSite=strict`, reducing CSRF protection.

**Files Fixed**:
- `/root/.openclaw/workspace/7zi-project/src/lib/api/utils.ts`

**Changes**:
- Changed `auth_token` cookie from `SameSite=lax` to `SameSite=strict`
- Changed `refresh_token` cookie from `SameSite=lax` to `SameSite=strict`
- Updated default cookie options to use `strict` policy
- Added explanatory comment about enhanced CSRF protection

**Impact**: Stronger CSRF protection; cookies only sent with same-site requests, preventing cross-site request forgery attacks.

---

- **After**: ✅ Low Risk (environment variable enforcement + strict CORS + strict cookies)

### 技术细节

**索引设计原理**:
- **复合索引选择**: `(agent_id, expires_at)` 支持同时按 agent 和过期时间查询
- **单列索引**: `roles(name)` 支持精确和模糊查询
- **布尔索引**: `roles(is_system)` 优化系统角色过滤
- **货币索引**: `agent_wallets(currency)` 支持多币种钱包管理

**与现有索引的关系**:
- v2 的 `idx_agent_tokens_expires` (仅 expires_at) 仍保留
- v3 的 `idx_agent_tokens_agent_expires` (agent_id + expires_at) 提供更精细的查询优化
- 两者互补，不冲突
