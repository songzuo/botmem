## 🔧 CI/CD 替代方案 - Gitea Actions

### ✅ 推荐方案: Gitea Actions

**研究完成时间**: 2026-03-08

**选择理由:**
1. ✅ 与现有 Gitea 仓库无缝集成
2. ✅ 语法兼容 GitHub Actions (迁移成本低)
3. ✅ 轻量级，资源占用低
4. ✅ 自托管，数据完全可控
5. ✅ 社区活跃，文档完善
6. ✅ **完全免费，无分钟限制**

### 对比分析

| 方案 | 成本 | 集成度 | 维护难度 | 推荐度 |
|------|------|--------|----------|--------|
| Gitea Actions | 免费 | ⭐⭐⭐⭐⭐ | 低 | ✅ 推荐 |
| GitHub Actions | 付费 | ⭐⭐⭐ | 中 | - |
| GitLab CI | 免费 | ⭐⭐⭐⭐ | 高 | - |
| Jenkins | 免费 | ⭐⭐⭐ | 高 | - |

### Pipeline 配置示例

```yaml
# .gitea/workflows/ci-cd.yml
name: CI/CD Pipeline

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  ci:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'npm'
      - run: npm ci
      - run: npm run lint
      - run: npm run type-check
      - run: npm run test:run

  deploy:
    needs: ci
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - run: npm run build
      - run: npm run start
```
