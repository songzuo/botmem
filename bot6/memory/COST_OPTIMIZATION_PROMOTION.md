### 1.4 存储优化建议（紧急）

**问题**: 磁盘使用率 82% (72GB/88GB)

**立即可行方案**:

```bash
# 1. 清理旧日志
find /var/log -type f -name "*.log" -mtime +30 -delete
find /var/log -type f -name "*.gz" -delete

# 2. 清理 Docker 未使用资源
docker system prune -af --volumes

# 3. 清理 npm 缓存
npm cache clean --force
rm -rf /root/.npm/_cacache

# 4. 清理旧构建产物
rm -rf /var/www/7zi/.next/cache/*
```

**预计释放空间**: 5-10GB

### 1.5 CDN 实施方案

**推荐**: Cloudflare 免费版

| 功能      | 免费版 | 付费版 ($20/月) |
| --------- | ------ | --------------- |
| 流量      | 无限   | 无限            |
| DDoS 防护 | ✅     | ✅ 增强         |
| SSL       | ✅     | ✅              |
| 缓存规则  | 3 条   | 不限            |
| 页面规则  | 3 条   | 不限            |
| WAF       | ❌     | ✅              |
| 图片优化  | ❌     | ✅              |

**实施步骤**:

1. 在 Cloudflare 添加 7zi.com 域名
2. 更新域名 DNS 服务器
3. 配置缓存规则：
   - `/_next/static/*` - 缓存 1 年
   - `/public/*` - 缓存 1 周
   - `/api/*` - 不缓存
4. 启用 Auto Minify (JS/CSS/HTML)
5. 启用 Brotli 压缩

**预期效果**:

- 服务器带宽减少 60-80%
- 全球访问延迟降低 30-50%
- SEO 评分提升（Core Web Vitals）

### 1.6 Redis 缓存方案

**部署方案**: 本地 Docker 运行 Redis

```yaml
# docker-compose.yml
services:
  redis:
    image: redis:7-alpine
    ports:
      - '6379:6379'
    volumes:
      - redis_data:/data
    command: redis-server --maxmemory 256mb --maxmemory-policy allkeys-lru
    restart: unless-stopped
```

**缓存策略**:

- API 响应缓存 5-15 分钟
- 页面缓存 1-5 分钟
- Session 存储

**预期效果**:

- API 响应时间减少 50-80%
- 数据库查询减少 60%
- 并发处理能力提升 3-5 倍
