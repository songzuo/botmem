### 核心问题分析

1. **WebSocket/Socket.io Mock 不完整**（最严重）
   - `useNotifications` (22/22 失败)、`websocket-store` (11/29 失败)、`collab-client` (15/38 失败)
   - setup.ts 中的 socket.io mock 基础但测试需要更精细的模拟

2. **手势/触摸测试超时**（useTouchGestures: 25/52 失败）
   - 120s 超时表明事件模拟器（jsdom）对 TouchEvent 支持有限
   - 建议：这类测试在 CI 中标记为 `@skip` 或降低并行度

3. **React act() 警告**
   - `Dashboard.test.tsx`、`ErrorBoundary.test.tsx` 有 act() 包装警告
   - 非致命但说明异步状态更新测试不标准

4. **异步语音提供商测试**
   - STTRouter 测试中 "should use fallback provider on error" 和 "should throw error if all providers fail" 各耗时 120s
   - 这是网络请求测试，不应在单元测试中真正发起 HTTP 调用

### 修复建议

| 优先级 | 建议 |
|--------|------|
| P0 | 增强 `src/test/setup.ts` 中的 socket.io mock，支持更多事件和状态转换 |
| P1 | 检查 `useNotifications` 测试的 `connect()` 时序，可能需要 `vi.useFakeTimers()` |
| P1 | `useTouchGestures` 测试建议标记为 `@skip` 或改为 e2e 测试 |
| P2 | 修复 `global-error-handler.test.ts` 全部失败问题（7/7） |
| P2 | 解决 `act()` 警告：确保异步状态更新被正确包装 |
| P3 | 优化 `STTRouter.test.ts` 的 provider 测试，避免真实网络调用 |

### 总结
项目 Vitest 配置正确，无类型错误。测试可以运行但有约 68 个测试失败，主要集中在 WebSocket mock 不完整和触摸手势测试两类。建议优先修复 socket.io mock，这会一次性解决多个测试套件的失败问题。
