### 分析目标
为 v1.5.0 的 `lib/` 层重构做准备，分析 agent 相关目录的混乱情况。

---

### 发现的问题

#### 1. 目录重复问题

| 重复组 | 旧位置 | 新位置 | 状态 |
|--------|--------|--------|------|
| agent 核心 | `src/lib/agent/` | `src/lib/agents/agent/` | ⚠️ 重复存在 |
| a2a 协议 | `src/lib/a2a/` | `src/lib/agents/a2a/` | ⚠️ 重复存在 |
| scheduler | `src/lib/agent-scheduler/` | `src/lib/agents/scheduler/` | ⚠️ 重复存在 |

#### 2. 当前使用情况

**`@/lib/agent`** (老路径，通过 legacy-agent-exports.ts 兼容):
- 已废弃，通过 `legacy-agent-exports.ts` 重导出到 `./agents/agent`
- 文件内容与 `agents/agent/` 几乎相同（diff 显示细微差异）

**`@/lib/a2a`** (老路径):
- 仍有 10+ 处使用：`app/api/a2a/*` 路由
- 文件内容与 `agents/a2a/` 相同（仅缺少 index.ts）

**`@/lib/agent-scheduler`** (独立模块):
- 仍有 2 处使用：`agent-dashboard/page.tsx`, `TeamStatus.tsx`
- 与 `agents/scheduler/` 内容有差异

---

### 重构方案建议

#### 方案：统一到 `src/lib/agents/`

```
src/lib/agents/
├── agent/           # 核心 Agent 操作 (auth, repository, wallet)
├── a2a/             # Agent-to-Agent 协议
├── scheduler/       # 调度系统
└── tools/           # 工具函数
```

#### 迁移步骤

1. **Phase 1: 更新导入路径**
   ```bash
   # a2a 路径迁移
   @/lib/a2a/* → @/lib/agents/a2a/*
   
   # scheduler 路径迁移  
   @/lib/agent-scheduler/* → @/lib/agents/scheduler/*
   ```

2. **Phase 2: 删除重复目录**
   ```bash
   rm -rf src/lib/agent/
   rm -rf src/lib/a2a/
   rm -rf src/lib/agent-scheduler/
   rm src/lib/legacy-agent-exports.ts
   ```

3. **Phase 3: 更新 tsconfig paths** (如果有的话)

#### 预计工作量
- **0.5 天** - 更新导入路径 (约 15-20 处)
- **0.5 天** - 测试验证 + 清理废弃文件

---

### 风险评估

| 风险 | 级别 | 缓解措施 |
|------|------|----------|
| 导入路径遗漏 | 低 | grep 全局搜索验证 |
| 测试失败 | 低 | 运行完整测试套件 |
| 运行时错误 | 中 | 分阶段迁移 + 验证 |

---

### 结论

lib/ 层存在明显的目录重复问题，需要将：
- `lib/agent/` → 已有兼容层，迁移成本低
- `lib/a2a/` → 需要更新 API 路由导入
- `lib/agent-scheduler/` → 需要更新 Dashboard 组件导入

**建议**: 在 v1.5.0 P0 阶段完成此重构，工作量 1 天足够。
