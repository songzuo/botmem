# Proactive Decision

主动决策和智能建议系统。

## 核心理念

不是被动等待指令，而是主动：
- **监控** - 持续关注系统状态
- **分析** - 识别潜在问题和机会
- **建议** - 提出优化方案
- **执行** - 自动或半自动实施决策

## 决策框架

### 1. 信息收集
```
监控指标 → 数据分析 → 状态评估
```

### 2. 问题识别
```
阈值检测 → 趋势分析 → 异常发现
```

### 3. 方案生成
```
问题定义 → 方案枚举 → 优劣评估
```

### 4. 执行决策
```
自动执行 / 请求确认 / 记录日志
```

## 主动检查项

### 系统健康
- [ ] 磁盘使用率 < 80%
- [ ] 内存使用率 < 90%
- [ ] CPU 负载 < 80%
- [ ] 关键服务运行中
- [ ] 日志无严重错误

### 项目状态
- [ ] Git 状态干净
- [ ] 依赖无安全漏洞
- [ ] 测试通过
- [ ] 构建成功
- [ ] 文档更新

### 安全检查
- [ ] 无可疑登录
- [ ] 防火墙正常
- [ ] SSL 证书有效
- [ ] 敏感数据加密

## 决策脚本

运行 `scripts/health_check.py` 进行全面检查：

```bash
python3 scripts/health_check.py
```

输出包括：
- 系统健康评分
- 问题列表
- 建议操作
- 优先级排序

## 自动化决策规则

### 高优先级（自动执行）
- 磁盘 > 90% → 清理缓存
- 内存 > 95% → 重启服务
- 服务崩溃 → 自动重启

### 中优先级（请求确认）
- 磁盘 > 80% → 建议清理
- 依赖过时 → 建议更新
- 证书即将过期 → 提醒续期

### 低优先级（记录日志）
- 性能下降 → 记录分析
- 使用趋势 → 统计报告

## 建议输出格式

```markdown
## 🔍 系统状态分析

### ⚠️ 发现问题
1. 磁盘使用率 85%（警告）
2. npm 依赖 3 个安全漏洞

### 💡 建议操作
1. [高优先级] 运行 `npm audit fix` 修复漏洞
2. [中优先级] 清理 node_modules 和缓存

### 📊 预测
- 3 天后磁盘可能达到 90%
- 建议尽快执行清理

是否执行建议操作？[Y/n]
```

## 最佳实践

1. **渐进式自动化** - 从通知开始，逐步增加自动执行
2. **明确边界** - 危险操作必须确认
3. **记录决策** - 保持决策日志便于审计
4. **学习优化** - 根据反馈调整决策规则
