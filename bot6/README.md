## 📖 项目介绍

**7zi** 是一个革命性的 AI 驱动团队管理平台，由 **11 位专业 AI 成员** 组成完整的组织架构。我们重新定义了团队协作的可能性 —— 不再是人类管理工具，而是 **AI 团队自主工作**，人类只需制定战略方向。

### 🌟 核心创新

- **🤖 AI 主管系统** - 智能任务分配与协调

- **📊 实时 Dashboard** - 透明化所有工作进展
- **🔄 自主决策** - 在授权范围内独立完成任务
- **🎯 目标驱动** - 专注于结果而非过程

## 🚀 快速开始

### 环境要求

- Node.js 22+
- pnpm 8+ (推荐) 或 npm 10+
- Git

### 本地运行

```bash
# 1. 克隆仓库
git clone https://github.com/songzuo/7zi.git
cd 7zi

# 2. 安装依赖
pnpm install
# 或
npm install

# 3. 启动开发服务器（无需配置环境变量即可运行基础功能）
pnpm dev
# 或
npm run dev

# 4. 访问应用
# 打开浏览器访问 http://localhost:3000
```

### 环境变量配置（可选）

创建 `.env.local` 文件以启用高级功能：

```bash
# 网站配置
NEXT_PUBLIC_SITE_URL=https://your-domain.com

# EmailJS（联系表单）
NEXT_PUBLIC_EMAILJS_PUBLIC_KEY=your_public_key
NEXT_PUBLIC_EMAILJS_SERVICE_ID=your_service_id
NEXT_PUBLIC_EMAILJS_TEMPLATE_ID=your_template_id

# Resend（邮件通知）
RESEND_API_KEY=xxx

# 告警通知
SLACK_WEBHOOK_URL=xxx
ALERT_EMAIL_RECIPIENTS=admin@example.com,ops@example.com

# Sentry（错误监控）
NEXT_PUBLIC_SENTRY_RELEASE=1.0.0
```

### 生产部署

```bash
# 构建
pnpm build

# 启动生产服务
pnpm start
```

### Docker 部署

```bash
# 构建镜像
docker build -t 7zi-team .

# 运行容器
docker run -p 3000:3000 --env-file .env 7zi-team
```

## 📖 新功能使用指南

### Portfolio 项目展示模块

```bash
# 访问项目展示页面
http://localhost:3000/portfolio

# API 端点
GET  /api/projects       # 获取项目列表
POST /api/projects       # 创建新项目
GET  /api/projects/:id   # 获取项目详情
```

**使用示例:**
```tsx
import { ProjectGrid, ProjectCard } from '@/components';

function PortfolioPage() {
  const { projects, loading } = useProjects();
  
  return (
    <ProjectGrid>
      {projects.map(project => (
        <ProjectCard key={project.id} project={project} />
      ))}
    </ProjectGrid>
  );
}
```

### Tasks AI 任务分配系统

```bash
# 访问任务管理页面
http://localhost:3000/tasks

# API 端点
GET    /api/tasks           # 获取任务列表
POST   /api/tasks           # 创建新任务
PUT    /api/tasks/:id       # 更新任务
DELETE /api/tasks/:id       # 删除任务
POST   /api/tasks/:id/assign # AI 智能分配
```

**使用示例:**
```tsx
import { TaskBoard, TaskCard, useTasks } from '@/components';

function TasksPage() {
  const { tasks, createTask, updateTask } = useTasks();
  
  const handleAssign = async (taskId: string) => {
    await fetch(`/api/tasks/${taskId}/assign`, { method: 'POST' });
  };
  
  return <TaskBoard tasks={tasks} onAssign={handleAssign} />;
}
```

### 日志系统 API

```bash
# API 端点
GET  /api/logs         # 获取日志列表
GET  /api/logs/:id     # 获取日志详情
POST /api/logs         # 创建日志条目
GET  /api/logs/export # 导出日志
```

**日志类型:**
- `task` - 任务执行日志
- `system` - 系统事件日志
- `error` - 错误日志
- `audit` - 审计日志
