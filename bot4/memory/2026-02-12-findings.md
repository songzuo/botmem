## 🔍 OpenClaw 配置能力分析

### 支持的 API 协议

| 协议类型 | 描述 | 是否支持 ARK |
|----------|--------|----------|
| `openai-completions` | OpenAI 聊天 | ❌ |
| `openai-responses` | OpenAI Responses | ❌ |
| `anthropic-messages` | Anthropic Claude | ❌ |
| `google-generative-ai` | Google Gemini | ❌ |
| `github-copilot` | GitHub Copilot | ❌ |
| `bedrock-converse-stream` | AWS Bedrock | ❌ |

**结论**: OpenClaw **不支持**火山引擎的 `ark-responses` 协议

### Fallback 机制

OpenClaw 已有 fallback 功能：
- 通过 `agents.defaults.model.primary` 设置主模型
- 通过 `agents.defaults.model.fallbacks` 设置备用模型列表
- 自动在主模型失败时切换到备用模型
- 支持多个 provider

## 💡 推荐解决方案

### 方案 1: 使用 MCP 服务器（推荐）

**原因**:
- ✅ 现成可用，立即可用
- ✅ 官方通过 MCP 协议支持
- ✅ 提供搜索和图片生成功能
- ✅ 无需修改 OpenClaw 配置

**步骤**:
1. 在 Claude Desktop 中配置 MCP 连接到 VolcEngine Search MCP Server
2. 使用火山引擎的搜索功能
3. 如需要图片生成，配置 VolcEngine Image MCP
4. 享受增强的 AI 功能

**优势**:
- 通过标准化 MCP 协议集成
- 支持 Claude Desktop 原生集成
- 开源可验证和自托管
- 功能立即可用

**局限**:
- 仅支持工具功能，不支持 LLM 聊天
- 无法通过 OpenClaw 配置调用 LLM 模型

---

### 方案 2: 部署本地代理服务

**原因**:
- ✅ 利用已找到的 ASR/TTS OpenAI 兼容方案
- ✅ 可作为本地 HTTP 服务运行
- ✅ 提供语音功能

**步骤**:
1. 使用 Docker 部署 volcengine-asr-openai 和 volcengine-tts-openai
2. 配置 OpenClaw 使用本地服务地址调用语音识别和合成
3. 需要自定义 provider 支持

**优势**:
- 提供完整的语音功能
- 可本地控制和管理
- 降低延迟

**局限**:
- 仅支持 ASR 和 TTS，不支持 LLM 聊天
- 需要额外的服务器资源和维护

---

### 方案 3: 优化现有 Fallback（短期）

**原因**:
- ✅ 立即可用
- ✅ 配置文件已支持 fallback
- ✅ 阿里云千问模型可用

**步骤**:
1. 修改 openclaw.json 配置
2. 添加更多阿里云千问模型到 fallback 列表
3. 测试 fallback 功能

**建议模型顺序**:
1. zai/glm-4.7 (主模型，智谱)
2. qwen-dashscope/qwen3-omni-flash (千问3多模态)
3. qwen-dashscope/qwen3-coder-plus (千问3 Coder)
4. qwen-coding/qwen3-coder-plus (千问3 Coder Plus)
5. qwen-dashscope/qwen3-turbo (千问3 Turbo)
6. qwen-dashscope/qwen-max (千问3 Max)

**优势**:
- 立即可用
- 所有模型使用 OpenAI 兼容协议
- 无需额外服务

**局限**:
- 不包含火山引擎的 LLM 模型
- 无法使用火山引擎的联网搜索等增强功能

---

### 方案 4: 开发 OpenClaw 插件（长期）

**原因**:
- 🚀 彻底支持自定义 API 协议
- 🚀 可能需要与 OpenClaw 团队合作

**步骤**:
1. 研究 OpenClaw 插件开发文档
2. 设计火山引擎 ARK 协议适配器
3. 实现自定义 provider
4. 向 OpenClaw 提交 PR 或提案

**优势**:
- 完整支持火山引擎 ARK 协议
- 可直接在配置中添加火山引擎模型
- 社区贡献，帮助其他用户

**挑战**:
- 开发复杂度高
- 需要深入理解 OpenClaw 内部架构
- 可能需要较长的审批和测试周期

## 📌 最终建议

### 短期方案 (立即可用)

1. **使用 MCP 服务器** (主要推荐)
   - 在 Claude Desktop 中配置 VolcEngine Search MCP Server
   - 享受火山引擎的搜索和图片生成功能
   - 适合需要工具功能的场景

2. **优化现有 Fallback 配置**
   - 添加更多阿里云千问模型作为备用
   - 确保主模型（GLM-4.7）余额不足时可以切换
   - 适合日常聊天和编程任务

### 中期方案 (如需要 LLM 功能)

1. **部署本地代理服务**
   - 使用 volcengine-asr-openai 和 volcengine-tts-openai
   - 提供语音识别和合成功能
   - 需要研究 OpenClaw 是否支持自定义 HTTP provider

2. **开发 OpenClaw 插件**
   - 向 OpenClaw 团队咨询或提交功能请求
   - 添加火山引擎 ARK 协议支持
   - 实现完整的 LLM fallback 功能
