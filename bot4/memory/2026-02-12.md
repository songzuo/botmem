## OpenClaw 大模型 Fallback 配置 - 火山引擎兼容性问题

### 请求
用户要求为 OpenClaw 添加大模型 Fallback 功能，包含以下火山引擎模型：
1. **千问3多模态** - https://dashscope.aliyuncs.com/compatible-mode/v1 (已有配置，阿里云)
2. **DS3.2 deepseek** - https://ark.cn-beijing.volces.com/api/v3/responses (火山引擎 ARK 协议)
3. **GLM4.7** - https://ark.cn-beijing.volces.com/api/v3/responses (火山引擎 ARK 协议)
4. **豆包1.8** - https://ark.cn-beijing.volces.com/api/v3/responses (火山引擎 ARK 协议)
5. **阿里云DashScope Coder Plus** - https://coding.dashscope.aliyuncs.com/v1 (已有配置)

### 发现的问题
**火山引擎 ARK 协议与 OpenAI 兼容协议不兼容**

火山引擎 ARK 协议请求格式：
```json
{
  "model": "deepseek-v3-2-251201",
  "stream": true,
  "tools": [{"type": "web_search", "max_keyword": 3}],
  "input": [{
    "role": "user",
    "content": [{"type": "input_text", "text": "今天有什么热点新闻"}]
  }]
}
```

OpenClaw 当前支持的 API 协议：
- `openai-completions` - OpenAI 兼容格式
- `openai-responses` - OpenAI Responses API
- `anthropic-messages` - Anthropic Claude 格式
- `google-generative-ai` - Google Gemini 格式
- `github-copilot` - GitHub Copilot 格式
- `bedrock-converse-stream` - AWS Bedrock 格式

**无法直接在配置中添加火山引擎的 `ark-responses` 协议**

### 当前 Fallback 配置
```
主模型: zai/glm-4.7 (智谱 GLM-4.7)
备用模型:
  - qwen-dashscope/qwen3-omni-flash (千问3多模态)
  - qwen-dashscope/qwen3-coder-plus (千问3 Coder Plus)
  - qwen-coding/qwen3-coder-plus (千问3 Coder)
  - qwen-coding/qwen3-max-2026-01-23 (千问3 Max)
```

### 技术限制
- OpenClaw 配置 schema 仅支持以下 API 协议类型，无法直接添加火山引擎 ARK 协议

## 📋 解决方案总结

### 方案 A: 使用 MCP 服务器 (推荐)

#### 优势
- ✅ 已有现成项目，可直接使用
- ✅ 通过 MCP 协议与 Claude Desktop 等应用集成
- ✅ 支持火山引擎的搜索和图片生成功能
- ✅ 无需修改 OpenClaw 配置
- ✅ 开源项目，可验证和自托管

#### 实施步骤
```bash
# 1. 部署 MCP 服务器
npm install -g volcengine-search-mcp-server
export VOLCENGINE_API_KEY=your_api_key
npx volcengine-search-mcp-server

# 2. 在 Claude Desktop 中配置 MCP 连接
# 在 Claude Desktop 设置中添加 MCP 服务器：
{
  "volcengine": {
    "command": "npx",
    "args": ["volcengine-search-mcp-server"]
  }
}

# 3. 使用火山引擎功能
# 现在 Claude Desktop 中可以直接使用火山引擎的搜索和图片生成功能
```

#### 局限
- ⚠️ 仅支持工具功能（搜索、图片生成），不支持 LLM 聊天
- ⚠️ 无法直接通过 OpenClaw 配置调用 LLM 模型
- ⚠️ 需要使用 Claude Desktop 的 MCP 连接功能

### 方案 C: 优化现有 Fallback 配置 (短期方案)

#### 优势
- ✅ 立即可用
- ✅ 配置文件已支持 fallback 机制
- ✅ 阿里云千问模型使用 OpenAI 兼容协议

#### 实施步骤
```json
// 在 openclaw.json 中优化 fallback 配置
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "zai/glm-4.7",
        "fallbacks": [
          "zai/glm-4.7",
          // 主模型重试一次
          "qwen-dashscope/qwen3-omni-flash",      // 千问3多模态
          "qwen-dashscope/qwen3-coder-plus",     // 千问3 Coder
          "qwen-dashscope/qwen3-coder-plus",     // 千问3 Coder
          "qwen-coding/qwen3-max-2026-01-23" // 千问3 Max
          // 可添加更多阿里云模型：
          // "qwen-dashscope/qwen-plus"
          // "qwen-dashscope/qwen-max"
          // "qwen-dashscope/qwen-turbo"
          // "qwen-coding/qwen-plus"
        ]
      }
    }
  }
}
```

#### 局限
- ⚠️ 不包含火山引擎的 LLM 模型
- ⚠️ 火山引擎的 LLM 功能无法通过 OpenAI 兼容协议使用
