### 错误描述

- **原因**: 看到类似文件名就随意使用
- **后果**: 7zi.com HTTPS 显示证书过期警告

## 📋 方法论落地状态

### 核心方法论

| 方法论 | 是否落地 | 问题 |
|--------|----------|------|
| **先调查，后操作** | ❌ 未落地 | 没有检查现有证书配置 |
| **继承原有环境** | ❌ 未落地 | 没有继承原有证书路径 |
| **迭代更新** | ❌ 未落地 | 直接替换而不是增量修改 |
| **验证结果** | ❌ 未落地 | 修改后没有验证证书 |
| **保留回滚方案** | ❌ 未落地 | 没有备份原有配置 |

## 🔧 具体落地措施

### 1. 先调查，后操作

**检查清单**：
- [ ] 现有配置文件是什么？
- [ ] 现有证书路径是什么？
- [ ] 现有证书有效期？
- [ ] 有哪些备份文件？

**命令示例**：
```bash
# 检查现有证书
openssl x509 -in <cert> -noout -dates

# 检查 Nginx 配置
grep -r 'ssl_certificate' /etc/nginx/

# 验证当前运行证书
echo | openssl s_client -connect domain:443 | openssl x509 -noout -dates
```

---

### 2. 继承原有环境

**原则**：
- 不删除原有配置文件
- 在原有配置基础上修改
- 保留原有文件路径

**示例**：
```bash
# ❌ 错误：直接覆盖
cp new.crt /path/to/cert.crt

# ✅ 正确：先备份，再替换
cp /path/to/cert.crt /path/to/cert.crt.bak
cp new.crt /path/to/cert.crt
```

---

### 3. 迭代更新

**原则**：
- 每次只改一个配置
- 改完验证，再改下一个
- 增量修改，不是全量替换

---

### 4. 验证结果

**检查清单**：
- [ ] 服务是否正常启动？
- [ ] 配置是否生效？
- [ ] 功能是否正常？
- [ ] 有无报错？

**命令示例**：
```bash
# 验证 Nginx 配置
nginx -t

# 验证证书
openssl s_client -connect domain:443

# 测试 HTTPS
curl -I https://domain/
```

---

### 5. 保留回滚方案

**原则**：
- 任何修改前先备份
- 备份文件带时间戳
- 保留最近 3 个版本

**示例**：
```bash
# 备份配置
cp nginx.conf nginx.conf.bak.$(date +%Y%m%d-%H%M%S)

# 回滚
cp nginx.conf.bak.20260307-093000 nginx.conf
systemctl reload nginx
```

## 📋 证书配置检查清单

### 修改前
- [ ] 检查现有证书路径
- [ ] 检查现有证书有效期
- [ ] 备份现有证书
- [ ] 备份 Nginx 配置

### 修改时
- [ ] 使用正确的证书文件
- [ ] 确保证书和密钥匹配
- [ ] 更新 Nginx 配置

### 修改后
- [ ] 测试 Nginx 配置 (`nginx -t`)
- [ ] 重启 Nginx (`systemctl restart nginx`)
- [ ] 验证证书 (`openssl s_client`)
- [ ] 测试 HTTPS (`curl -I`)

---

## 🎯 关键原则

> **任何操作，都必须继承原有的环境和条件，然后再做迭代和更新。**

1. **不删除**原有配置
2. **先备份**再修改
3. **增量修改**不是全量替换
4. **验证结果**确保修改生效
5. **保留回滚**方案
