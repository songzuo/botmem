## 6. 服务日志摘要

### Nginx 错误日志 (最近 50 条，排除 timeout)

**主要问题分类:**

1. **SSL 握手失败 (CRITICAL)**
   - 错误: `SSL_do_handshake() failed (SSL: error:0A00006C:SSL routines::bad key share)`
   - 来源 IP: 212.102.40.218, 116.62.143.63, 120.48.251.219, 39.105.14.55 等
   - 频率: 持续出现，多个 IP
   - **判断:** 恶意扫描/漏洞探测

2. **ACME SSL 证书验证失败**
   - 请求: `/.well-known/acme-challenge/*`
   - 来源: 多个 IP 尝试获取 SSL 证书
   - **判断:** Let's Encrypt 证书续期请求，非关键

3. **dating.7zi.com:8082 大量 404**
   - 大量请求寻找旧 PHP 路径 (exchangerateuserconfig!get.action, leftDao.php 等)
   - 来源: 36.255.220.149
   - **判断:** 恶意扫描探测历史漏洞

4. **WordPress 探测**
   - wp-login.php, admin.php, license.php 等
   - 来源: 45.86.202.x, 91.217.249.x 等
   - **判断:** 常见自动化扫描攻击

## 8. 建议行动

### 立即建议

1. **fail2ban 配置** - 针对持续 SSL 握手失败的 IP 设置自动封禁
2. **日志清理** - 清理旧的 Nginx error.log，减小磁盘压力
3. **监控** - 设置磁盘使用率告警 (建议 >80%)

### 可选优化

1. **Swap 配置** - 添加 2-4GB Swap 分区
2. **SSL 配置** - 检查 SSL 协议和加密套件配置
3. **清理 PM2** - 删除不需要的 ex-7zi/export-7zi 应用
