# 配置文件管理与API路由系统

## 任务1: API路由自动切换系统

### 目标
- 适用于Picoclaw和OpenClaw
- 探针自动探测
- 快速切换
- 先测试1-2台，稳定后推广

### 实现方案
```
┌─────────────────────────────────────────────────────┐
│              Unified API Gateway                     │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐             │
│  │ Picoclaw│  │OpenClaw │  │  Other  │             │
│  └────┬────┘  └────┬────┘  └────┬────┘             │
│       │            │            │                    │
│       └────────────┼────────────┘                    │
│                    ▼                                  │
│         ┌──────────────────┐                          │
│         │   API Router     │                          │
│         │  (This Server)   │                          │
│         └────────┬─────────┘                          │
│                  │                                     │
│    ┌────────────┼────────────┐                       │
│    ▼            ▼            ▼                        │
│ ┌──────┐  ┌────────┐  ┌────────┐                    │
│ │Probe │  │ Health │  │ Stats  │                    │
│ │Agent │  │Monitor │  │Collector│                    │
│ └──┬───┘  └────┬───┘  └───┬────┘                    │
│    │           │           │                         │
│    └───────────┼───────────┘                         │
│                ▼                                      │
│  ┌────────────────────────────────────────┐          │
│  │         Provider Pool                   │          │
│  │  - volcengine (火山引擎)                │          │
│  │  - alibaba (阿里云)                    │          │
│  │  - siliconflow (硅基流动)               │          │

│  │  - groq, deepseek, openrouter          │          │
│  └────────────────────────────────────────┘          │
└─────────────────────────────────────────────────────┘
```

### 关键点
1. **Coding Plan端点识别** - 避免计费异常
   - 火山引擎: `ark.cn-beijing.volces.com/api/coding` ✅
   - 阿里云: `coding.dashscope.aliyuncs.com` ✅
   
2. **健康检测** - 每10秒探测
3. **快速切换** - 失败立即切换
4. **测试策略** - 先bot6，稳定后推广

## 任务2: 配置文件梳理

### 需要收集
```
每台机器:
├── openclaw.json (当前配置)
├── openclaw.json.bak* (历史备份)
├── picoclaw.json (Picoclaw配置)
├── credentials/
│   ├── telegram/ (电报Token)
│   └── *.json
└── 频道配置
```

### 分析内容
1. **API Keys清单** - 统一管理
2. **端点URL检查** - 识别coding plan
3. **频道配置** - Telegram、DingTalk等
4. **插件状态** - 识别冲突
