# 智能体每日工作流程

## 核心原则

**简单有效，充分利用OpenClaw功能**

## 每日工作流程

### 1. 早晨：启动和检查 (09:00 - 09:30)

#### 1.1 系统状态检查
```bash
# 检查系统状态
openclaw status

# 查看其他智能体的工作成果
ls -la /root/.openclaw/workspace/memory/
```

#### 1.2 工作成果阅读
- 阅读其他智能体的工作记录
- 检查项目文档更新
- 了解当前项目状态

### 2. 上午：主要任务执行 (09:30 - 12:00)

#### 2.1 任务执行
```markdown
# 今日任务
- [ ] 创建测试工程师智能体配置文件
- [ ] 建立项目管理文档结构
- [ ] 开始实施简单的协作机制
```

#### 2.2 工作记录
```markdown
# 工作成果记录
- 完成了测试工程师智能体配置文件
- 建立了项目管理文档结构
- 实现了基础的协作功能
```

### 3. 下午：协作和审查 (13:30 - 16:00)

#### 3.1 成果审查
- 审查其他智能体的工作成果
- 提供反馈和建议
- 评估工作质量

#### 3.2 协作讨论
```markdown
# 协作讨论
## 主题：系统架构优化

**技术专家**：建议使用微服务架构
**项目经理**：担心开发时间和复杂度
**测试工程师**：支持，但需要完善测试策略

**决策**：采用模块化架构，预留扩展接口
```

### 4. 傍晚：总结和规划 (16:00 - 17:00)

#### 4.1 每日总结
```markdown
# 每日工作总结
- 完成了3个主要任务
- 审查了2个智能体的工作成果
- 参与了1个协作讨论
- 记录了1份工作总结
```

#### 4.2 明日计划
```markdown
# 明日工作计划
- [ ] 创建文档专家智能体配置文件
- [ ] 优化协作流程
- [ ] 开始测试系统功能
```

## 工作记录规范

### 记录格式
```markdown
# 工作成果记录
## 任务完成
- [任务名称]：详细描述和成果
- [任务名称]：详细描述和成果

## 问题解决
- [问题描述]：解决方案和效果

## 决策记录
- [决策主题]：决策过程和结果

## 学习收获
- [知识点]：学习和应用过程

## 改进建议
- [改进点]：优化建议和措施
```

### 文件名规范
```
memory/YYYY-MM-DD.md
```

## 协作规范

### 反馈机制
- 积极反馈：详细说明优点和值得学习的地方
- 建设性批评：具体的改进建议
- 问题讨论：客观分析和解决方案

### 决策流程
1. 提出问题
2. 收集意见
3. 分析评估
4. 达成共识
5. 执行和跟踪

## 效率优化

### 工具使用
- **使用`memory_search`查找相关信息**
- **使用`read`快速阅读文档**
- **使用`write`记录工作成果**
- **使用`exec`执行任务**

### 时间管理
- 每天分配固定时间阅读工作成果
- 及时记录工作过程
- 定期总结和规划

---

**简单就是力量。通过遵循这个简单的流程，我们可以实现高效的智能体团队协作。**
