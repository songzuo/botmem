# HEARTBEAT.md - 心跳检查清单

作为总指挥官，我需要定期检查其他机器人的动态。

## 每次心跳检查

### 1. 观察更新 (Observer)
- [ ] 扫描 botmem_temp/ 下所有机器人的最新 memory 文件
- [ ] 记录新发现的任务、问题、进展

### 2. 检查是否有紧急情况 (Firefighter)
- [ ] 是否有 P0/P1 级别的问题需要立即处理
- [ ] 是否有机器人需要紧急救援

### 3. 检查健康状态 (HealthMonitor)
- [ ] 集群节点是否都在线
- [ ] 资源使用是否正常

### 4. 更新观察记录
- [ ] 记录本次观察时间
- [ ] 更新观察状态

## 每日循环

### 早上 (08:00 UTC)
- 全面扫描所有机器人
- 生成每日观察报告

### 中午 (12:00 UTC)
- 检查上午的更新
- 评估是否需要介入

### 下午 (17:00 UTC)
- 检查下午的更新
- 协调子代理处理问题

### 晚上 (21:00 UTC)
- 生成每日总结
- 规划明日工作

## 关键触发条件

需要立即派遣子代理的情况：
- 机器人遇到无法解决的问题
- 发现重大安全隐患
- 任务严重滞后
- 发生服务中断
