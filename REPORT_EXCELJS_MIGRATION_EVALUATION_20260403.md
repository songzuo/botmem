## API 兼容性对比

### 基本功能对比

| 功能 | exceljs | xlsx (SheetJS) | 兼容性 | 备注 |
|------|---------|----------------|--------|------|
| **读取 Excel** | `new Workbook().xlsx.readFile()` | `XLSX.readFile()` | ⚠️ 中等 | API 完全不同，需重写 |
| **写入 Excel** | `workbook.xlsx.writeBuffer()` | `XLSX.writeFile()` | ⚠️ 中等 | API 完全不同，需重写 |
| **创建工作簿** | `new Workbook()` | `XLSX.utils.book_new()` | ⚠️ 中等 | 语法不同 |
| **创建工作表** | `workbook.addWorksheet()` | `XLSX.utils.aoa_to_sheet()` | ⚠️ 中等 | 数据结构不同 |
| **样式支持** | ✅ 完整支持 | ⚠️ 有限/需付费 | ❌ 不兼容 | xlsx 免费版样式受限 |
| **公式支持** | ✅ 完整支持 | ⚠️ 读取/写入受限 | ⚠️ 中等 | xlsx Pro 更完善 |
| **图表支持** | ✅ 原生支持 | ❌ 不支持 | ❌ 不兼容 | 需要其他库或 Pro 版本 |
| **条件格式** | ✅ 支持 | ❌ 不支持 | ❌ 不兼容 | - |
| **流式处理** | ✅ 支持大文件流 | ⚠️ 内存加载 | ❌ 不兼容 | 大文件场景关键差异 |
| **类型定义** | ✅ 内置 TypeScript | ✅ @types/xlsx | ✅ 兼容 | 均有良好支持 |

### 数据结构对比

**exceljs 风格**:
```typescript
const workbook = new Workbook();
const sheet = workbook.addWorksheet('Sheet1');
sheet.columns = [
  { header: 'Name', key: 'name' },
  { header: 'Age', key: 'age' }
];
sheet.addRow({ name: 'John', age: 30 });
```

**xlsx 风格**:
```typescript
const workbook = XLSX.utils.book_new();
const data = [['Name', 'Age'], ['John', 30]];
const sheet = XLSX.utils.aoa_to_sheet(data);
XLSX.utils.book_append_sheet(workbook, sheet, 'Sheet1');
```

## 建议

### 场景一：项目当前未使用 exceljs

**✅ 推荐**: 如果需要 Excel 功能，根据需求选择：

| 需求场景 | 推荐方案 |
|----------|----------|
| 纯数据读写（无样式） | **xlsx** - 体积小，性能好 |
| 需要样式支持 | **exceljs** - 功能完整 |
| 需要图表 | **exceljs** - 原生支持 |
| 大文件处理（>100MB） | **exceljs** - 流式 API |
| 仅导出简单报表 | **xlsx** + 后处理 |

### 场景二：如果未来考虑迁移

**⚠️ 不建议迁移**，如果：

1. 使用了样式功能（单元格颜色、字体、边框）
2. 需要图表生成
3. 处理超大 Excel 文件（流式读写）
4. 依赖条件格式

**✅ 可以考虑迁移**，如果：

1. 仅做纯数据导入/导出
2. 不需要任何样式
3. 文件较小（<10MB）
4. 想要减少 bundle 体积
