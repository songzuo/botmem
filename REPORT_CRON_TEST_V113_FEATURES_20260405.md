## 测试框架使用

### Vitest 配置

测试使用 Vitest 框架编写，主要使用的测试工具：

- `describe()`: 测试套件组织
- `it()`: 单个测试用例
- `expect()`: 断言
- `beforeEach()`: 每个测试前的准备
- `afterEach()`: 每个测试后的清理
- `vi.useFakeTimers()`: 模拟定时器（用于自动化引擎定时触发器测试）
- `vi.runAllTimersAsync()`: 异步执行所有定时器

### 测试最佳实践

1. **测试隔离**: 使用 `beforeEach` 和 `afterEach` 确保测试独立
2. **Mock 使用**: 使用 `vi` 进行函数模拟和定时器控制
3. **断言明确**: 使用 `expect` 进行明确的断言
4. **错误覆盖**: 测试正常路径和错误路径
5. **边界测试**: 测试极限值和异常情况
