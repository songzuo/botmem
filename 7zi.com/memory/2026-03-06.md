# 2026-03-06 记忆

## 核心架构理解

### OpenClaw 运行模式
- **7zi.com**: 本地模式 (`mode: local`, `bind: loopback`)
- **不是配对模式**: Gateway 不连接到远程机器
- **Mesh 网络独立**: 通过 WebSocket (19000) 和 REST API (19001) 通信

### 三种自动投票机制

1. **Hub 轮巡投票** (技术层面)
   - 每 15 秒 heartbeat 检测
   - 自动判定离线成员
   - 自动故障转移

2. **Agents 自主投票** (AI 决策)
   - Agents 自己讨论、自己决策
   - 有分歧时自动发起投票
   - 无需人类参与

3. **Agents + 人类混合投票** (人机协作)
   - Agents 和人类都参加群聊
   - 意见不同时自动投票
   - 自动出结果、自动执行

### 高可用架构
- **Hub 至少 2 台**: 7zi.com (Active) + bot5 (Standby)
- **Registrar 至少 2 台**: bot4 (Active) + bot6 (Standby)
- **自动故障切换**: 一台故障时自动切换到另一台

## 关键 Bug 修复

### 消息发送功能修复 (12:20 UTC)

**问题**: 消息发送成功但不显示，不被记录到历史

**根本原因**: Hub 模式下，`client.sendChat()` 通过 WebSocket 发送，但 Hub 没有连接到自己，消息没被添加到 history

**修复方案**: 修改 `/opt/claw-mesh/mesh.mjs`，Hub 模式下手动将发送的消息添加到 history：

```javascript
// 修复后
client.sendChat(body.message || "");
if (client.isHub) {
  history.push({
    from: cfg.memberName,
    fromRole: cfg.memberRole,
    type: "chat",
    channel: cfg.channel,
    content: body.message,
    timestamp: Date.now()
  });
}
```

**测试**: ✅ 消息现在正确显示和记录

**文件**: `/opt/claw-mesh/mesh.mjs`, `/var/www/mesh-chat/index.html`
