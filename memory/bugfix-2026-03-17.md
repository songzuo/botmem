### 2. 🟠 中等：未处理的Promise Rejection - check_all.js

**位置**: `/root/.openclaw/workspace/commander/check_all.js:119`

**问题描述**:
使用 `child_process.exec()` 执行外部命令时，没有处理 Promise rejection 或回调错误。

**问题代码**:
```javascript
exec('node /root/.openclaw/workspace/commander/check_health.js', (error, stdout, stderr) => {
  if (error) {
    console.error('健康检查执行失败:', error);
    process.exit(1);
  }
  if (stderr) {
    console.error(stderr);
  }
  console.log(stdout);
});
// ❌ 缺少 .catch() 或对 error 的完整处理
```

**潜在问题**:
- 如果 `check_health.js` 执行失败，程序会调用 `process.exit(1)` 退出
- 但在退出前没有清理 SSH 连接 (`conn.end()` 在更早的回调中已调用)
- 在某些异步场景下可能导致资源泄漏

**修复建议**:
```javascript
exec('node /root/.openclaw/workspace/commander/check_health.js', (error, stdout, stderr) => {
  if (error) {
    console.error('健康检查执行失败:', error);
    // 确保已退出，不要再调用 process.exit()
    process.exit(1);
    return;
  }
  if (stderr) {
    console.error(stderr);
  }
  console.log(stdout);
}).on('error', (execError) => {
  console.error('健康检查启动失败:', execError);
  process.exit(1);
});
```

### 3. 🟠 中等：错误处理不完整 - check_all.js

**位置**: `/root/.openclaw/workspace/commander/check_all.js`

**问题描述**:
多个 `conn.exec()` 回调中的 stream 错误处理不完整。

**问题代码示例**:
```javascript
stream2.on('error', (streamErr) => {
  console.error('Stream2 error:', streamErr.message);
  // ❌ 只是记录错误，但没有清理或退出
});

stream2.on('close', () => {
  // 继续执行下一个检查，即使上一个失败了
  conn.exec('ss -tlnp | grep -E "80:|443:|3000:|517[0-9]:"', ...);
});
```

**潜在问题**:
- 如果 stream 发生错误，程序仍会继续执行后续检查
- 这可能导致不一致的状态或错误的报告

**修复建议**:
```javascript
let stream2Error = false;
stream2.on('error', (streamErr) => {
  console.error('Stream2 error:', streamErr.message);
  stream2Error = true;
});

stream2.on('close', () => {
  if (stream2Error) {
    conn.end();
    process.exit(1);
    return;
  }
  // 继续正常流程
  conn.exec('ss -tlnp | grep -E "80:|443:|3000:|517[0-9]:"', ...);
});
```

### 5. 🟡 轻微：潜在的内存泄漏 - check_health.js

**位置**: `/root/.openclaw/workspace/commander/check_health.js`

**问题描述**:
缓存文件可能在某些异常情况下未被清理。

**问题代码**:
```javascript
async function checkSingleSite(conn, site, useCache = true) {
  // ...
  const cached = getCachedResult(site.url);
  if (cached && cached.result) {
    console.log(colorize(`  📦 ${site.name}: 使用缓存数据`, 'blue'));
    return { ...healthResult, ...cached.result, cached: true };
  }
  // ...
  const result = await withTimeout(withRetry(checkFn, 1, 1000), 15000, ...);
  if (isHealthy) {
    setCachedResult(site.url, finalResult);
  }
  // ❌ 如果检查失败且不是 healthy，缓存不会更新
  // ❌ 但后续调用可能会继续使用过期的缓存
}
```

**潜在问题**:
- 如果检查失败，旧的缓存数据可能持续被使用
- 缓存过期时间（5分钟）可能不够灵活

**修复建议**:
- 为失败的结果也设置较短的缓存时间（如30秒）
- 或者在缓存中标记失败状态
