## 二、P0 问题详细分析

### 1. globals.css 暗色模式样式错误 🔴

**位置:** `src/app/globals.css` 第 45-47 行

**当前代码:**
```css
/* 暗色模式下的全局样式 */
.dark body {
  color: var(--color-gray-900);
  background-color: var(--color-gray-50);
}
```

**问题:** 这是完全错误的！在暗色模式下：
- `color: var(--color-gray-900)` = 深灰色文字 ❌ (应该是浅色)
- `background-color: var(--color-gray-50)` = 浅灰色背景 ❌ (应该是深色)

正确应该是使用 `--color-text-primary` 和 `--color-background` CSS 变量，或直接使用亮色值。

**影响:**
- 暗色模式下所有页面文字变成深灰色，背景是浅灰色
- 文字几乎不可见，用户体验极差
- 违反 WCAG 对比度要求

**修复建议:**
```css
.dark body {
  color: var(--color-gray-50);  /* 浅色文字 */
  background-color: var(--color-gray-900);  /* 深色背景 */
}
```

### 2. Button 组件移动端触控区域不足 ⚠️

**位置:** `src/components/ui/Button.tsx`

**问题:** 并非所有 Button 尺寸都满足 44x44px 最小触控区域要求

| 尺寸 | 当前 min-h/min-w | 是否达标 |
|------|-----------------|----------|
| xs | 可能 < 36px | ❌ |
| sm | 可能 < 40px | ❌ |
| md | 44px ✅ | ✅ |
| lg | 48px ✅ | ✅ |
| xl | 52px ✅ | ✅ |

**影响:**
- 移动端 xs/sm 按钮点击困难
- 不符合 Apple iOS Human Interface Guidelines (44x44pt)
- 影响用户操作准确性

**修复建议:**
```tsx
sizeStyles = {
  xs: 'px-2.5 py-1 text-xs gap-1 min-h-[36px] min-w-[36px]',
  sm: 'px-3 py-1.5 text-sm gap-1.5 min-h-[40px] min-w-[40px]',
  md: 'px-4 py-2 text-sm gap-2 min-h-[44px] min-w-[44px]',  // iOS 推荐
  lg: 'px-5 py-2.5 text-base gap-2 min-h-[48px] min-w-[48px]',
  xl: 'px-6 py-3 text-lg gap-2.5 min-h-[52px] min-w-[52px]',
}
```

### 3. Modal 可访问性缺陷 🔴

**位置:** `src/components/ui/Modal.tsx`

**当前实现:**
- ✅ ESC 键关闭
- ✅ Body 滚动锁定
- ✅ 焦点保存/恢复
- ✅ 关闭按钮 aria-label

**缺失的可访问性功能:**
| 功能 | 状态 | 说明 |
|------|------|------|
| `role="dialog"` | ❌ 缺失 | 应添加 `role="dialog"` |
| `aria-modal="true"` | ❌ 缺失 | 告诉屏幕阅读器这是模态 |
| `aria-labelledby` | ❌ 缺失 | 指向标题元素 ID |
| `aria-describedby` | ❌ 缺失 | 指向内容区域 ID |
| 焦点陷阱 | ⚠️ 不完整 | 只 focus modal，但用户可 Tab 出范围 |
| 标题 ID | ❌ 缺失 | 标题元素没有 `id` |

**影响:**
- 屏幕阅读器用户无法识别模态框
- 键盘用户可能 Tab 逃出模态框
- 无法通过标题理解模态框内容
- 不符合 WCAG 2.1 AA 标准

**修复建议:**
```tsx
// 头部添加 ID
{title && (
  <h2 id="modal-title" className="text-xl font-semibold text-gray-900">
    {title}
  </h2>
)}

<div
  ref={modalRef}
  role="dialog"
  aria-modal="true"
  aria-labelledby="modal-title"
  aria-describedby="modal-content"
  // ...
>
  {/* 内容添加 ID */}
  <div id="modal-content" className="px-6 py-4">
    {children}
  </div>
```

**焦点陷阱建议 (使用 react-focus-trap 或手动实现):**
```tsx
// 在 useEffect 中实现完整的焦点陷阱
// 监听 Tab 键，将焦点限制在模态框内
```

### 4. `text-muted-foreground` CSS 类定义 🟡

**状态:** 已定义但位置需明确

**发现:**
- 在 Tailwind 构建输出中存在 `.text-muted-foreground{color:var(--muted-foreground)}`
- `theme.css` 中 `--muted-foreground` 有正确定义
- `src/lib/theme/theme-config.ts` 中定义了 `textMuted` 变量

**问题:**
- 该类在 `globals.css` 中没有明确显式定义
- 完全依赖 Tailwind CSS 构建输出的自动注入
- 如果单独查看 `globals.css`，无法确认该类定义

**建议:**
- 如果继续使用 Tailwind，这没问题
- 如果不使用 Tailwind，需要在 `globals.css` 中显式定义
