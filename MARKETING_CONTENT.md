## 🎬 场景 2：深度功能 - RBAC 权限管理

**【开场】**

**主持人**：在企业应用中，权限管理是重中之重。7zi 的 RBAC（基于角色的访问控制）系统如何保障你的数据安全？让我来演示。

**【步骤 1：了解 RBAC 架构】**

**主持人**：首先，让我们理解 7zi 的 RBAC 架构：

**[屏幕：展示 RBAC 架构图]**

```
用户 (User)
  ↓
角色 (Role)
  ↓
权限 (Permission)
  ↓
资源 (Resource)
```

**主持人**：

- **用户**：系统的实际使用者
- **角色**：用户的身份标签（管理员、编辑、访客）
- **权限**：可执行的操作（读、写、删除、审核）
- **资源**：被保护的对象（任务、文档、设置）

**【步骤 2：创建角色】**

**主持人**：现在，让我为团队创建三个角色：

**[屏幕：角色管理界面]**

**主持人**：点击"新建角色"：

**角色 1：管理员**

```json
{
  "name": "管理员",
  "description": "拥有系统全部权限",
  "permissions": [
    "system:*", // 系统管理
    "user:*", // 用户管理
    "task:*", // 任务管理
    "settings:*" // 设置管理
  ]
}
```

**角色 2：项目经理**

```json
{
  "name": "项目经理",
  "description": "负责项目任务管理",
  "permissions": ["task:read", "task:create", "task:update", "task:delete", "dashboard:view"]
}
```

**角色 3：访客**

```json
{
  "name": "访客",
  "description": "只读权限",
  "permissions": ["task:read", "dashboard:view"]
}
```

**主持人**：点击"保存"，三个角色就创建完成了。

**【步骤 3：分配用户角色】**

**主持人**：现在，让我给团队成员分配角色：

**[屏幕：用户管理界面]**

**主持人**：选择用户"张三"，分配"项目经理"角色：

**[屏幕：角色分配界面]**
