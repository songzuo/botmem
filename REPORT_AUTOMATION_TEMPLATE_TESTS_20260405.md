## ⚠️ 已知限制

### 1. Cron 表达式验证器不支持 `*/N` 语法

**影响范围**: `template_data_sync` 模板
**cron 表达式**: `0 */6 * * *` （每 6 小时）

**问题描述**:
- 当前 `RuleValidator.isValidCron()` 不支持 `*/N` 这种简化的间隔语法
- 验证器仅支持：`*`、具体值、范围（如 `1-5`）、步长（如 `*/6`）

**解决方案**:
- 方案 1：更新验证器以支持 `*/N` 语法（推荐）
- 方案 2：将 `0 */6 * * *` 改为 `0 0,6,12,18 * * *`
- 方案 3：使用 `interval` 类型替代 `cron` 类型

**优先级**: 中等
**建议**: 在下一个版本中增强 cron 验证器

### 2. API URL 验证器不支持相对路径

**影响范围**: 多个模板中的 API 动作
**相对路径示例**: `/api/health`, `/api/monitoring/alert`

**问题描述**:
- 当前 `RuleValidator.isValidUrl()` 使用 `new URL()` 验证
- 仅支持完整 URL（如 `https://example.com/api/health`）
- 不支持相对路径（如 `/api/health`）

**解决方案**:
- 方案 1：更新验证器以支持相对路径（推荐）
- 方案 2：在验证前拼接基础 URL
- 方案 3：使用白名单策略（允许特定前缀的相对路径）

**优先级**: 中等
**建议**: 在下一个版本中增强 URL 验证器
