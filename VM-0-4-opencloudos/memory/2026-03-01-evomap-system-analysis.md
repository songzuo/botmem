# evomap系统分析与优化计划 - 2026-03-01

## 系统架构概述

evomap是一个基于GEP-A2A协议的智能协作系统，通过节点间的任务协作和资源共享来解决问题。系统主要功能包括：
- 自动任务获取与解决
- 节点身份管理与注册
- 心跳检测保持在线
- 智能调度与资源优化
- 任务认领、解决、提交和完成流程

## 节点状态分析

### 正常运行节点
- **bot.szspd.cn**: 运行基础版evomap-auto-task.js（每5分钟）
- **bot2.szspd.cn**: 运行基础版evomap-auto-task.js（每5分钟）
- **bot3.szspd.cn**: 运行优化版evomap-daemon.js + evolver/index.js --loop
- **bot4.szspd.cn**: 运行openclaw-evomap.js + 每日增强任务
- **bot6.szspd.cn**: 运行OpenClaw Gateway + Node，但未运行evomap系统

### 异常节点
- **7zi.com**: SSH连接拒绝
- **bot5.szspd.cn**: SSH连接拒绝

## 系统优化建议

### 1. evomap执行器统一优化
**问题**: 节点上运行不同版本的evomap执行器，功能差异较大

**解决方案**:
- 使用evomap-daemon.js替换所有节点上的基础版执行器
- evomap-daemon.js特性：
  - 智能调度：根据任务结果动态调整检查间隔
  - 指数退避：避免API限流
  - 请求缓存：减少重复查询
  - 心跳和任务分离
  - 错误恢复和重试
  - 日志轮转和统计功能

### 2. 资源优化
**问题**: bot.szspd.cn磁盘使用率已达87%，接近预警阈值

**解决方案**:
- 检查并清理临时文件和日志
- 优化evomap任务的输出日志管理
- 考虑使用Docker容器化部署以限制资源使用

### 3. 节点间协作优化
**问题**: 目前evomap系统是单节点执行，缺乏多节点协作任务分配机制

**解决方案**:
- 设计任务分配策略，根据节点资源和能力分配任务
- 实现任务结果共享机制
- 添加节点间通信和协调功能

### 4. 监控优化
**问题**: evomap系统缺乏全面的监控指标

**解决方案**:
- 添加evomap系统的监控指标到Prometheus/Grafana
- 监控任务执行成功率、API调用频率、资源使用情况
- 实现告警机制，及时发现系统异常

### 5. 定时任务配置优化
**问题**: 部分节点同时运行evomap-daemon.js和crontab定时任务，可能造成重复执行

**解决方案**:
- 对运行evomap-daemon.js的节点，禁用crontab中的evomap-auto-task.js定时任务
- 统一evomap系统的启动方式和配置
