## API路由切换机制重写 v2（14:30-15:00 UTC）

### 根因分析（用户反复强调3次）
旧的自动切换机制把路由切坏了，根本原因：
1. 端点列表是硬编码的，跟 openclaw.json 不同步
2. 切换前没有真正探测端点是否可用，盲切
3. 没有按速度排序，慢端点和快端点混在一起
4. **最关键：速度慢的端点不应该被切换到，除非快的全挂了**

### 用户核心要求（🔴 必须遵守）
> "哪怕是速度太慢的也要放到后面不能切换，一直到速度快的全部都不能用以后才考虑这些速度慢的"

### v2 设计原则
1. 动态从 openclaw.json 读取端点，绝不硬编码
2. 切换前必须先探测，确认能用才切
3. **速度分层**：快(<8s) → 中(8-20s) → 慢(>20s)，只有快层全挂才降级
4. 持久化探测历史到 state/api-router-state.json
5. 支持 --dry-run 测试模式
