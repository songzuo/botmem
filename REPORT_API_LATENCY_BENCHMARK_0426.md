## 七、性能问题识别

### 7.1 主要瓶颈

1. **搜索端点 (`/api/search`)**
   - 问题: P95 = 623ms，部分请求超过 1 秒
   - 原因: 无缓存，每次查询触发数据库 LIKE
   - 影响: 用户体验，搜索等待时间长

2. **数据库 JOIN 查询**
   - 问题: P95 = 123ms，复杂查询可能更长
   - 原因: 缺少复合索引，部分查询全表扫描
   - 影响: 数据加载慢，列表页卡顿

3. **缓存命中率低 (22% miss)**
   - 问题: 健康检查外大量端点未缓存
   - 原因: 缓存策略不完整
   - 影响: 重复计算，响应延迟

4. **Volcengine API 超时**
   - 问题: 0.8% 超时率
   - 原因: 网络不稳定时无重试
   - 影响: AI 推理失败，任务中断

### 7.2 慢查询识别

| 查询类型 | 预估延迟 | 频率 | 优先级 |
|---------|---------|------|--------|
| 搜索 LIKE '%xxx%' | 300-600ms | 高 | P1 |
| 多表 JOIN 无索引 | 100-200ms | 中 | P1 |
| 聚合 COUNT 全文扫描 | 150-300ms | 中 | P2 |
| Session 验证无缓存 | 50-100ms | 高 | P2 |

## 八、优化建议

### 8.1 P1 优化 (立即执行)

#### 1. 搜索端点缓存
```typescript
// 在 /api/search/route.ts 中添加
const cacheKey = `search:${query}:${page}:${limit}`
const cached = cacheManager.get(cacheKey)
if (cached) return cached

const results = await db.search(...)
cacheManager.set(cacheKey, results, 300000) // 5分钟
```

**预估效果:**
- P95: 623ms → 180ms (降低 71%)
- 缓存命中率: 0% → 75%

#### 2. 数据库索引优化
```sql
-- 用户表复合索引
CREATE INDEX idx_users_email_status ON users(email, status);

-- 菜谱表搜索索引
CREATE INDEX idx_recipes_title_search ON recipes(title);

-- 评分表复合索引
CREATE INDEX idx_ratings_recipe_user ON ratings(recipe_id, user_id);
```

**预估效果:**
- JOIN 查询 P95: 123ms → 45ms (降低 63%)
- LIKE 查询 P95: 300ms → 80ms (降低 73%)

### 8.2 P2 优化 (本周内)

#### 3. 连接池调优
```typescript
// prisma/client.ts
const prisma = new PrismaClient({
  datasources: {
    db: {
      url: process.env.DATABASE_URL + '?connection_limit=15&pool_timeout=10'
    }
  }
})
```

**预估效果:**
- 并发能力: +50%
- 连接等待: -60%

#### 4. API 重试机制 (Volcengine)
```typescript
async function withRetry(fn, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn()
    } catch (err) {
      if (i === maxRetries - 1) throw err
      await sleep(1000 * Math.pow(2, i)) // 指数退避
    }
  }
}
```

**预估效果:**
- 超时率: 0.8% → 0.1%

### 8.3 P3 优化 (计划中)

#### 5. 状态端点缓存

#### 6. Redis 分布式缓存
